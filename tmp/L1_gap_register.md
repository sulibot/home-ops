# Level 1 Gap Register

## Purpose

Track the meaningful `CMMC Level 1` gaps that remain between the current `level_1` artifact set and an attestation-ready package.

## Scope

- `FCI` boundary definition
- Ownership and accountability
- Practice-to-evidence traceability
- Readiness packaging

## Gaps

| Level | Gap Name | Why It Matters | Existing Files That Partially Cover It |
| --- | --- | --- | --- |
| L1 | Scope statement missing | Level 1 readiness cannot be defended without a clearly defined `FCI` boundary | `level_1/*.md`, handbook Level 1 sections |
| L1 | Owner matrix missing | Policies and evidence cannot be sustained without named accountable owners | `level_1/*.md` |
| L1 | Evidence index missing | Assessment readiness depends on mapping every practice to proof | `level_1/*.md`, handbook evidence discussion |
| L1 | Gap assessment tracker missing | There is no repeatable way to record and close deficiencies | handbook Level 1 gap analysis content |
| L1 | Readiness review package missing | Leadership and technical reviewers need one place to review readiness | all current Level 1 artifacts |

## Definition of Done

- Each Level 1 gap has an owner.
- Each gap is mapped to a prep or deliverable file.
- Each gap is either closed or tracked with target dates.
